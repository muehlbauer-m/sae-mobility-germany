# Data Directory

This directory is intentionally empty. The analysis pipeline expects the following data structure:

```
data/
├── raw/
│   ├── MiD2017_Regional_Personen.csv       # MiD 2017 person-level data
│   ├── MiD2017_Regional_Wege.csv           # MiD 2017 trip-level data
│   ├── census/
│   │   └── census_merged_2017.rds          # Generated by census_data_acquisition.r
│   └── shapes/
│       └── vg2500_01-01.gk3.shape/         # BKG district boundaries shapefile
│           └── vg2500/VG2500_KRS.shp
├── temp/
│   ├── inkar_2017_districts.rds            # Generated by inkar_data_acquisition.r
│   └── inkar_variable_metadata.rds         # Generated by inkar_data_acquisition.r
├── data_cleaning/                          # Generated by Step 1 (data_cleaning.r)
└── output/                                 # Generated by the pipeline
    ├── direct/
    │   └── df_HTH_perskm_analytical.rds    # Direct estimates (Step 5)
    ├── aggregates/
    │   └── df_mid_covariates_combined.rds  # Aggregated MiD covariates (Step 4)
    ├── variables/
    │   ├── phase1_lasso_results.rds        # LASSO variable selection (Step 6)
    │   └── phase2_sparseR_results_*.rds    # sparseR results (Step 7)
    └── models/
        ├── phase1_prepared_data.rds        # Prepared data (Step 6)
        ├── phase3_cv_results.rds           # CV results (Step 8)
        └── fh_model_final.rds             # Final Fay-Herriot model (Step 8)
```

## Data Sources

### MiD 2017 (Mobilitat in Deutschland)

The MiD 2017 survey microdata are not publicly available. Access requires an application to the German Federal Ministry for Digital and Transport (BMDV). Eligible applicants include:

- Commissioners of regional in-depth samples (for traffic policy, planning, or science)
- Universities or research institutions (for independent scientific research)
- Other third parties (for publicly commissioned or funded research projects)

For more information: https://www.mobilitaet-in-deutschland.de

### Destatis Census Tables

District-level census tables are acquired programmatically from GENESIS-Online and Regionaldatenbank Deutschland via the `restatis` R package. The script `R/data_preparation/census_data_acquisition.r` handles this automatically. API keys are required (free registration at https://www-genesis.destatis.de).

### INKAR Regional Indicators

INKAR (Indikatoren und Karten zur Raum- und Stadtentwicklung) data are acquired via the `bonn` R package from the BBSR open data API. The script `R/data_preparation/inkar_data_acquisition.r` handles this automatically.

### District Boundaries Shapefile

Administrative boundaries (VG2500) are available from the German Federal Agency for Cartography and Geodesy (BKG) at https://gdz.bkg.bund.de.
